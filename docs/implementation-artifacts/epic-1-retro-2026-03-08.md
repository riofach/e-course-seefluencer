1. A simple and stable implementation is more valuable than premature complexity.
2. Shared surfaces like the navbar have outsized impact on perceived product quality.
3. Small UX state refinements can materially improve clarity and trust.
4. Technical conventions should be made explicit early to reduce rework in later epics.

## Deep Story Analysis Synthesis

### Common Struggles

- Import alias consistency (`~/` vs `@/`) was a recurring source of friction.
- Auth/session architecture decisions had broad downstream impact.
- React 19 form/action conventions required some learning during implementation.
- Validation rigor improved over the epic and benefited from review feedback.

### Recurring Review Feedback

- Keep `ActionResponse<T>` usage consistent.
- Use `cn()` consistently.
- Preserve `~/` import alias conventions.
- Maintain defense-in-depth for auth and route protection.
- Keep UI feedback states explicit: loading, success, validation, and error.

### Breakthrough Moments

- Locking the auth/session direction simplified RBAC and logout behavior.
- Treating DB state as source of truth for profile display improved correctness.
- Learning transfer from earlier stories improved later story quality.
